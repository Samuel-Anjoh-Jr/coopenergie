Do not skip phases. Do not reorder tasks within a phase.

How to Use This Guide
Copilot Agent Mode Setup
Open GitHub Copilot Chat in VS Code (or your IDE). Switch to Agent mode — not Inline Chat, not Ask mode. Agent mode allows Copilot to create files, run terminal commands, and modify multiple files autonomously without pausing for permission on each step.
Open VS Code in the root of your monorepo (cd coopenergie).
Open Copilot Chat panel.
Select "Agent" from the mode dropdown (not "Ask" or "Edit").
Paste the first task prompt and press Enter.
Wait for full completion — watch the terminal and file tree.
When done, paste the next prompt.
You can queue all prompts in a session and walk away.

Every prompt in this guide is written to be self-contained. Copilot will always know the full context it needs from the prompt alone — no prior conversation context required between tasks.

CRITICAL: Never run two prompts simultaneously. Always wait for a task to fully complete (no more terminal activity, all files saved) before starting the next. Concurrent edits will corrupt the codebase.

Checking Completion
A task is complete when:
No more activity in the Copilot terminal panel
No error dialogs open
Any bun install or bun run commands have exited cleanly
The file tree shows all expected new files

If a Task Fails
Read the error message in the terminal.
If it's a dependency conflict: run bun install in the relevant app directory and retry.
If it's a TypeScript error: re-queue the same prompt with "Fix any TypeScript errors in the files just created." appended.
If it's a network/timeout: retry the same prompt — Copilot is idempotent for file creation.

Task 0.4  Root utility scripts
  ⬤  COPILOT PROMPT — Task 0.4
Do everything autonomously. Do not ask for confirmation.
 
TASK: Create 3 utility scripts in the scripts/ folder.
 
--- FILE 1: scripts/sync-env.mjs ---
Reads the root .env file and copies its content to:
- apps/web/.env.local
- apps/backend/.env
- apps/mobile/.env
 
Use Node.js fs module. If a target directory does not exist yet, skip it with a console.log.
 
--- FILE 2: scripts/dev-with-cleanup.js ---
Uses child_process to run concurrently:
- bun run --cwd apps/web dev
- bun run --cwd apps/backend dev (only if apps/backend/src/main.ts exists)
- bun run run –cwd apps/mobile dev
Handles SIGINT/SIGTERM to cleanly kill child processes.
If apps/backend does not exist yet, just run web only.
 
--- FILE 3: scripts/clean.js ---
Removes only build artifacts — NOT node_modules.
Remove these directories if they exist (use rmSync with recursive+force):
- apps/web/.next
- apps/web/out
- apps/backend/dist
- apps/mobile/.expo
- contracts/artifacts
- contracts/cache
 
Then kill running processes:
try { execSync('pkill -f "bun run"', { stdio: 'ignore' }); } catch {}
try { execSync('pkill -f "next dev"', { stdio: 'ignore' }); } catch {}
try { execSync('pkill -f "nest start"', { stdio: 'ignore' }); } catch {}
console.log('🛑 Stopping all node processes...')
try {
  if (process.platform === 'win32') {
    execSync('taskkill /F /IM node.exe /T 2>nul  exit 0', { shell: true, stdio: 'inherit' })
    execSync('taskkill /F /IM bun.exe /T 2>nul  exit 0', { shell: true, stdio: 'inherit' })
  } else {
    execSync('pkill -f node  true', { shell: true, stdio: 'inherit' })
    execSync('pkill -f bun  true', { shell: true, stdio: 'inherit' })
  }
  console.log('✅ Processes stopped\n')
} catch {
  console.log('⚠️  No processes to kill\n')
}
 
Print "Build artifacts cleared. node_modules preserved." when done.

Task 2.2  GasRelayer.sol — meta-transaction relayer
  ⬤  COPILOT PROMPT — Task 2.2
Do everything autonomously. Do not ask for confirmation.
 
TASK: Create the GasRelayer.sol smart contract.
 
Create file: contracts/src/GasRelayer.sol
 
This is an EIP-2771 compatible meta-transaction forwarder.
 
Requirements:
- Owner-controlled: only the contract owner (the platform relayer wallet) can call execute()
- Nonce tracking: each user address has a nonce to prevent replay attacks
- Signature verification: verifies that the meta-transaction was signed by the stated 'from' address
- Expiry: transactions have a deadline timestamp parameter
- Whitelist: only whitelisted target contracts (CooperativeVault addresses) can be called
 
Interface:
struct ForwardRequest {
  address from;
  address to;
  uint256 nonce;
  uint256 deadline;
  bytes data;
}
 
function execute(ForwardRequest calldata req, bytes calldata signature) external returns (bool, bytes memory)
function getNonce(address from) external view returns (uint256)
function addWhitelisted(address target) external  // onlyOwner
function removeWhitelisted(address target) external  // onlyOwner
 
Use OpenZeppelin Ownable and ECDSA for the signature verification.

--- withdrawals.service.ts ---
 
Method: createWithdrawalProposal(adminUserId, cooperativeId, dto)
  dto: { amountXAF, reason, destinationType, recipientPhone?,
         recipientOperator?, recipientBankName?, recipientBankAccount?,
         recipientName }
  a. Validate: user is COOP_ADMIN
  b. Validate: cooperative.confirmedBalanceXAF >= amountXAF (cannot withdraw more than balance)
  c. Create Proposal { type: WITHDRAWAL, title: `Retrait: ${amountXAF} FCFA`, description: reason }
  d. Create WithdrawalRequest linked to the proposal
  e. Call relayerService.relayCreateProposal() on-chain
  f. Return { proposal, withdrawalRequest }
 
Method: getEligibleVoters(cooperativeId)
  Returns members who have at least 1 CONFIRMED contribution in this cooperative.
  Query: memberships JOIN contributions WHERE cooperativeId AND status=CONFIRMED
  Returns: array of { userId, amountContributed }
 
Method: evaluateWithdrawalThreshold(proposalId)
  a. Get the proposal's cooperativeId
  b. Load CooperativeSettings.withdrawalThreshold (or PlatformSettings.withdrawalThresholdDefault)
  c. Load quorumMin from PlatformSettings.withdrawalQuorumMinVotes
  d. Count eligible voters via getEligibleVoters()
  e. Count YES votes from eligible voters only:
     SELECT count(*) FROM votes WHERE proposalId AND choice=true
     AND userId IN (SELECT userId FROM eligible voters)
  f. Count NO votes from eligible voters
  g. totalEligibleVoted = yesVotes + noVotes
  h. If totalEligibleVoted < quorumMin: return { approved: false, reason: "quorum not reached" }
  i. yesPercent = (yesVotes / eligibleVoterCount) * 100
  j. If yesPercent >= threshold: return { approved: true, yesPercent }
  k. Else: check if it is mathematically impossible to reach threshold (remaining voters cannot change outcome)
     If so: return { approved: false, resolved: true }
  l. Else: return { approved: false, resolved: false } — still collecting votes
 
Method: processWithdrawalOnVote(proposalId)
  Called after every vote on a WITHDRAWAL proposal:
  a. Call evaluateWithdrawalThreshold()
  b. If approved: update Proposal to APPROVED, WithdrawalRequest to APPROVED
     Then: call disbursementService.disburse(withdrawalRequestId)
  c. If resolved but not approved: update to REJECTED
 
--- disbursement.service.ts ---
 
Method: disburse(withdrawalRequestId)
  a. Load WithdrawalRequest with cooperative
  b. Update status to DISBURSING
  c. Generate reference: `COOP-WD-${cooperativeId.slice(0,8)}-${Date.now()}`
  d. Call CamPay Transfer API:
     POST ${CAMPAY_BASE_URL}/transfer/
     Headers: Authorization: Token ${CAMPAY_TRANSFER_API_KEY}
     Body (for MoMo):
     {
       "amount": withdrawalRequest.amountXAF.toString(),
       "currency": "XAF",
       "to": withdrawalRequest.recipientPhone,
       "operator": withdrawalRequest.recipientOperator,  // "MTN" or "ORANGE"
       "description": `Retrait coopérative ${cooperative.name}`,
       "external_reference": reference
     }
  e. On CamPay success:
     - Update WithdrawalRequest: status=DISBURSED, campayReference, disbursedAt
     - Decrement cooperative.confirmedBalanceXAF by amountXAF
     - Call relayerService.relayFundsReleased() on-chain (if BLOCKCHAIN_ENABLED)
     - Write AuditLog
     - Publish GraphQL event: withdrawal.disbursed
  f. On CamPay failure:
     - Update status=FAILED, failureReason=campayError
     - Revert Proposal status to PENDING (allow retry vote)
     - Notify cooperative admin via email

   import { initializeApp, getApps } from "firebase/app"
   import { getMessaging, getToken, onMessage } from "firebase/messaging"
 
   const firebaseConfig = {
     apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY,
     authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
     projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID,
     storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
     messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID,
     appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID,
   }
 
   export const app = !getApps().length ? initializeApp(firebaseConfig) : getApps()[0]
 
   export async function requestNotificationPermission(): Promise<string | null> {
     if (typeof window === "undefined") return null
     if (!("Notification" in window)) return null
     const permission = await Notification.requestPermission()
     if (permission !== "granted") return null
     const messaging = getMessaging(app)
     const token = await getToken(messaging, {
       vapidKey: process.env.NEXT_PUBLIC_FIREBASE_VAPID_KEY,
       serviceWorkerRegistration: await navigator.serviceWorker.getRegistration()
     })
     return token
   }
 
   export function onForegroundMessage(callback: (payload: any) => void) {
     if (typeof window === "undefined") return () => {}
     const messaging = getMessaging(app)
     return onMessage(messaging, callback)
   }

   import * as Notifications from "expo-notifications"
   import * as Device from "expo-device"
   import Constants from "expo-constants"
 
   export async function registerForPushNotifications(): Promise<string | null> {
     if (!Device.isDevice) {
       console.warn("Push notifications only work on physical devices")
       return null
     }
 
     const { status: existingStatus } = await Notifications.getPermissionsAsync()
     let finalStatus = existingStatus
 
     if (existingStatus !== "granted") {
       const { status } = await Notifications.requestPermissionsAsync()
       finalStatus = status
     }
 
     if (finalStatus !== "granted") return null
 
     const projectId = Constants.expoConfig?.extra?.eas?.projectId
     const token = await Notifications.getExpoPushTokenAsync({ projectId })
     return token.data  // format: ExponentPushToken[xxxxxxxx]
   }
 
   export function configurePushHandlers() {
     // How to handle notifications while app is in foreground
     Notifications.setNotificationHandler({
       handleNotification: async () => ({
         shouldShowAlert: true,
         shouldPlaySound: true,
         shouldSetBadge: true,
       }),
     })
   }
 
4. Create apps/mobile/lib/notifications/use-push-notifications.ts:
   A hook that:
   a. Calls configurePushHandlers() once on mount
   b. Calls registerForPushNotifications()
   c. On success: calls POST /users/device-token { token, platform: "IOS" or "ANDROID" }
      Determine platform: Platform.OS === "ios" ? "IOS" : "ANDROID"
   d. Stores the token in MMKV ("push_token")
   e. Sets up notification response handler:
      Notifications.addNotificationResponseReceivedListener(response => {
        const data = response.notification.request.content.data
        if (data.type === "CONTRIBUTION") router.push("/(dashboard)/contributions")
        if (data.type === "PROPOSAL") router.push("/(dashboard)/proposals")
        if (data.type === "WITHDRAWAL") router.push("/(dashboard)/proposals")
      })
   f. Returns { pushToken, notificationsEnabled }
 
5. Call usePushNotifications() in the root dashboard layout:
   apps/mobile/app/(dashboard)/_layout.tsx
   Import and call the hook at the top level of the layout component.

   const pushToken = storage.getString("push_token")
   if (pushToken) {
     await api.delete("/users/device-token", { token: pushToken })
     storage.delete("push_token")
   }
